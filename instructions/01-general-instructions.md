# General User Instructions (Simulation Project)

## Goal
Use these instructions when requesting any change in this project.

## Request Format
1. **Objective**: What should change?
2. **Scope**: Which files/features are in scope?
3. **Constraints**: What must not change?
4. **Validation**: Which commands must pass?
5. **Output**: Short summary or detailed report.

## Rules
- Keep changes minimal and focused.
- Do not break existing simulation behavior.
- Do not modify unrelated files.
- Keep UI consistent with existing style.
- Update tests when behavior changes.

## Definition of Done
- Requested change implemented.
- No regressions in existing behavior.
- Tests pass.
- Coverage/report commands run when requested.
